**Workflow structure:**

The workflow is NOT an agent — it does not have YAML frontmatter with `name/tools/color`. It uses `<purpose>` and `<step>` blocks like execute-plan.md does.

**`<purpose>` section:**
The review-team workflow orchestrates the review pipeline after each plan completes. It receives inputs from the `review_team_gate` step: SUMMARY.md path, phase identifier, and plan identifier. Phase 2 implements validation and sanitization. Phase 3 adds reviewer spawning. Phase 4 adds synthesis and routing.

**`<inputs>` section:**
Document the three inputs this workflow receives from the review_team_gate step:
- `SUMMARY_PATH` — path to the current plan's SUMMARY.md (e.g., `.planning/phases/01-xxx/01-02-SUMMARY.md`)
- `PHASE_ID` — phase identifier (e.g., `01-extension-scaffold-gsd-integration`)
- `PLAN_NUM` — plan number (e.g., `02`)

These are passed by the review_team_gate step which says: "Pass: current SUMMARY.md path, phase identifier, plan identifier."

**Step 1 — `<step name="validate_team" priority="first">`:**

Read `.planning/TEAM.md` and validate it has at least one valid reviewer role.

A valid role requires ALL THREE of:
1. A `## Role:` section header
2. A YAML code block (triple-backtick yaml) containing a `name:` field
3. At least one item under the `**What this role reviews:**` heading

Validation logic (for the executing Claude to follow):
- Read `.planning/TEAM.md` (double-check it exists even though review_team_gate already confirmed — defensive)
- Split content on `## Role:` headers
- For each section after the first split (the preamble before the first role):
  - Look for a fenced YAML block containing `name:` — extract the value as role_name
  - Check the section contains `**What this role reviews:**` with at least one list item below it
  - If valid: add role_name to the valid roles list
  - If invalid: log a warning naming the specific section (e.g., "WARNING: Role section 'Security Auditor' missing YAML name: field — skipped")

If zero valid roles found after checking all sections:
- Output a structured error block:
```
## REVIEW PIPELINE HALTED

**Error:** TEAM.md exists but contains zero valid reviewer roles.

Each role requires:
1. A `## Role:` section header
2. A YAML code block with a `name:` field
3. At least one review criteria item under `**What this role reviews:**`

Run `/gsd:new-reviewer` to create your first reviewer role.
```
- HALT the pipeline. Do NOT proceed to sanitization. This satisfies TEAM-03.

If one or more valid roles found:
- Store the role count and role names for later use (Phase 3 will use them for reviewer spawning)
- Log: `Review Team: {N} reviewer role(s) found in TEAM.md: {comma-separated role names}`
- Proceed to sanitize step

**Step 2 — `<step name="sanitize">`:**

Construct the ARTIFACT.md output path following the GSD per-plan artifact convention:
```
ARTIFACT_PATH=".planning/phases/${PHASE_ID}/${PHASE_PADDED}-${PLAN_NUM}-ARTIFACT.md"
```
Where PHASE_PADDED is the zero-padded phase number extracted from PHASE_ID (e.g., "01" from "01-extension-scaffold-gsd-integration").

Spawn the sanitizer agent via Task():
```
Task(
  subagent_type="gsd-review-sanitizer",
  prompt="
    <objective>
    Sanitize the execution summary into a clean artifact description.
    Strip all executor reasoning. Preserve all observable facts.
    Write the artifact to disk.
    </objective>

    <inputs>
    Summary path: ${SUMMARY_PATH}
    Artifact output path: ${ARTIFACT_PATH}
    Phase: ${PHASE_ID}
    Plan: ${PLAN_NUM}
    </inputs>

    <execution_context>
    @~/.claude/get-shit-done-review-team/agents/gsd-review-sanitizer.md
    </execution_context>
  "
)
```

After the sanitizer returns:
- Verify ARTIFACT.md exists at the expected path:
  ```bash
  [ -f "${ARTIFACT_PATH}" ] && echo "ARTIFACT EXISTS" || echo "ARTIFACT MISSING"
  ```
- If ARTIFACT.md is missing: halt pipeline with error "Sanitizer completed but ARTIFACT.md not found at ${ARTIFACT_PATH}"
- If ARTIFACT.md exists: log `Review Team: ARTIFACT.md written to ${ARTIFACT_PATH}`
- Parse the sanitizer's return for strip count and section count (from the SANITIZATION COMPLETE block)
