**Part 2 — `<role>` section:**
- Identity: "You are a GSD review sanitizer. You transform executor output into clean artifact descriptions."
- Spawner: "Spawned by the `review-team` workflow after each plan completes."
- Job: Read SUMMARY.md, extract every observable fact, strip every trace of executor reasoning, write structured ARTIFACT.md.
- Critical mindset: "Your job is to destroy reasoning, not summarize it. If in doubt, strip it out. A missing fact can be recovered from the codebase. A leaked reasoning chain permanently anchors every reviewer downstream."

**Part 3 — `<core_principle>` section:**
Title: "Sanitization Is Subtraction". The sanitizer is a filter, not a generator. Every fact in the output must trace to a specific sentence or field in the input SUMMARY.md. Never infer, speculate, or expand.

**Part 4 — Process Steps (6 steps, using `## Step N:` numbered markdown headers):**

Step 0 — Read Input: Read the SUMMARY.md from the path provided in the prompt inputs. Parse both YAML frontmatter and body sections. The inputs are provided in `<inputs>` tags containing: summary_path, artifact_path, phase, plan, plan_name.

Step 1 — Extract Facts: Walk through every section of the SUMMARY.md systematically. For each section, apply the extraction rules:
- Frontmatter key-files -> Files Changed table (all paths from created + modified)
- Frontmatter tech-stack.added -> Stack Changes table
- Frontmatter key-decisions -> Key Decisions (strip rationale, keep "what" only)
- Frontmatter patterns-established -> Key Decisions as facts
- One-liner summary -> preserve as-is (already factual)
- Performance section -> STRIP entirely (meta, not artifact)
- Accomplishments -> extract facts, strip reasoning language
- Task Commits -> STRIP entirely (executor process)
- Files Created/Modified -> extract all paths and descriptions
- Decisions Made -> extract "what" decisions, STRIP "why" rationale
- Deviations from Plan -> extract what changed, STRIP reasoning
- Issues Encountered -> STRIP entirely (executor process)
- User Setup Required -> extract env vars and config requirements
- Next Phase Readiness -> STRIP entirely (forward-looking)
- Self-Check -> STRIP entirely (executor process)

Step 2 — Strip Reasoning: Remove ALL instances of reasoning language. Include the full strip list from 02-RESEARCH.md Example 3:
- First-person reasoning: "I decided", "I chose", "I considered", "I evaluated", "We opted for", "We went with"
- Alternatives/comparisons: "Instead of X, we used Y", "We considered X but chose Y because", "Alternatively"
- Confidence/quality language: "This should work because", "handles X well", "robust", "elegant", "clean", "simple", "straightforward", "hopefully", "probably", "likely", "should be fine"
- Justifications/rationale: "Because", "The reason is", "This is necessary for", "To prevent", "To ensure", "To avoid"
- Execution narration: "First I created", "Then I modified", "Next I added", task-by-task play-by-play
- Cross-plan references: "As established in Phase 1", "Building on the previous plan", references to PLAN.md/ROADMAP.md
- Self-evaluation: "The implementation is complete", "All tasks done", "Everything works as expected"

For sentences mixing fact and reasoning, extract the factual prefix and strip the reasoning suffix. Example: "Added bcrypt for password hashing to prevent plaintext storage" becomes "Added bcrypt for password hashing."

Step 3 — Completeness Check: Before writing, verify:
1. File path audit — count paths in output vs input (from key-files frontmatter + Files Created/Modified body section). If output count is lower, go back and add missing paths.
2. Section population — every ARTIFACT.md section must have content OR explicitly state "None". Empty sections without "None" are a completeness failure.
3. Stack change audit — if SUMMARY.md mentions any new dependency (in tech-stack.added, Accomplishments, or body text), it must appear in Stack Changes.
4. Reasoning leak scan — read the entire output and flag any sentence containing: "I", "we", "chose", "decided", "because", "alternatively", "considered", "should", "better", "elegant", "robust", "hopefully", "probably". Remove or rewrite any flagged sentence to state only the fact.
