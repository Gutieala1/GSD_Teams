Create `D:/GSDteams/workflows/agent-dispatcher.md` as a GSD workflow file using the standard
`purpose / inputs / step` structure with XML step blocks. The file is the source copy; install.sh
copies it to `~/.claude/get-shit-done-review-team/workflows/agent-dispatcher.md`.

The workflow has four steps in this exact order:

**Step 1: `check_bypass_and_config` (priority="first")**

Read the bypass flag from the current PLAN.md. The PLAN.md path is derived from the `plan_path`
input (passed by the calling review_team_gate step). If `plan_path` is not provided, derive it
from `summary_path` by replacing `-SUMMARY.md` with `-PLAN.md`.

Check for bypass flag using bash:
```bash
SKIP_REVIEW=$(grep -m1 "^skip_review:" "$PLAN_PATH" 2>/dev/null | awk '{print $2}' || echo "false")
```

If `SKIP_REVIEW` is `"true"`: log `Agent Dispatcher: skipped (agent creation plan — bypass active)`
and return immediately. Do NOT proceed to subsequent steps.

Read `AGENT_STUDIO_ENABLED` from `CONFIG_CONTENT` (already in scope from calling workflow):
```bash
AGENT_STUDIO_ENABLED=$(echo "$CONFIG_CONTENT" | jq -r '.workflow.agent_studio // false')
```

If `AGENT_STUDIO_ENABLED` is not `"true"`: log `Agent Dispatcher: agent_studio disabled — no-op`
and return immediately. (The v1 review_team path is handled directly in review_team_gate, not
by the dispatcher.)

**Step 2: `read_and_parse_team_md`**

Read `.planning/TEAM.md` using the Read tool. Extract:
- `version` from the YAML frontmatter (default: 1 if absent — numeric, not string)
- `roles` list from the frontmatter

For each role slug in the roles list, find its `## Role: {slug}` section in the TEAM.md body.
Extract the YAML config block between triple-backtick yaml fences in that section.
Parse fields: `name`, `focus`, `mode`, `trigger`, `tools`, `output_type`, `commit`,
`commit_message`, `scope`.

Apply `normalizeRole(role, version)` to every role. This shim triggers when `version < 2`
(numeric comparison — NOT string comparison):
```
if version < 2:
  role.mode             = role.mode             ?? "advisory"
  role.trigger          = role.trigger          ?? "post-plan"
  role.tools            = role.tools            ?? []
  role.output_type      = role.output_type      ?? "findings"
  role.commit           = role.commit           ?? false
  role.commit_message   = role.commit_message   ?? null
  role.scope            = role.scope            ?? {}
  role.scope.allowed_paths  = role.scope.allowed_paths  ?? []
  role.scope.allowed_tools  = role.scope.allowed_tools  ?? []
```

Result: `normalized_roles` — a list where every role has all 8 fields populated.

**Step 3: `filter_by_trigger`**

Filter `normalized_roles` to keep only roles where `role.trigger == trigger_type` (the input
`trigger_type`, e.g. `"post-plan"`).

If the filtered list is empty:
  Log: `Agent Dispatcher: no agents configured for trigger '{trigger_type}' — continuing`
  Return immediately. Do NOT call review-team.md. Do NOT spawn any Task().
  This is DISP-02: zero spawns, zero latency.

If filtered list is non-empty: group by `role.mode` into `advisory_roles` and `autonomous_roles`.

**Step 4: `route_by_mode`**

Handle advisory roles:
- If `advisory_roles` is non-empty:
  Call the review pipeline via @-reference:
  ```
  @~/.claude/get-shit-done-review-team/workflows/review-team.md
  ```
  Pass the same inputs currently received: `SUMMARY_PATH`, `PHASE_ID`, `PLAN_NUM`.
  review-team.md reads TEAM.md itself — do NOT pass a filtered role list or a `roles:` parameter.
  The dispatcher does not replicate review-team.md logic.

Handle autonomous roles (stub for Phase 9):
- If `autonomous_roles` is non-empty:
  For each autonomous role:
    Log: `Agent Dispatcher: autonomous agent '{role.name}' — autonomous execution not yet
    implemented (Phase 9). Skipping.`
  Do NOT spawn any Task(). Do NOT crash. This is a safe, informative no-op.

**Inputs block (document all inputs):**
- `trigger_type`: string — the trigger context ("post-plan", "pre-plan", "post-phase", "on-demand")
- `summary_path`: string — path to current SUMMARY.md (e.g. `.planning/phases/07-agent-dispatcher/07-01-SUMMARY.md`)
- `phase_id`: string — phase identifier (e.g. `"07-agent-dispatcher"`)
- `plan_num`: string — plan number (e.g. `"01"`)
- `plan_path`: string (optional) — path to current PLAN.md for bypass flag extraction

Do NOT include pre-plan or post-phase hook wiring — that is Phase 9 scope. The dispatcher
accepts those trigger types but only the post-plan calling point is wired in Phase 7.
